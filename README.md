## About

I work on practical software tasks where the goal is simple: make the broken thing reproducible, make the fix small, and leave the reviewer with enough context to trust it.

My strongest lanes are Python, TypeScript, browser/workflow automation, API integrations, data cleanup, and open-source bug fixes.

## Working Style

```text
1. Reproduce the issue
2. Find the smallest useful fix
3. Add or update tests when they reduce risk
4. Keep changes narrow and reviewable
5. Leave a clear handoff for the next person
```
